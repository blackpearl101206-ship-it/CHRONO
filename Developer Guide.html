02

Database & Data Structure

Frontend-only apps still deserve a real schema. Treat localStorage like a single-table database — one key, one JSON array. Every task is a plain object with a stable id, timestamps as "HH:MM" strings, and a boolean completion flag.

Task schema

// One task record
{
  "id":          "t_ab12cd34",        // unique — nanoid or random36
  "taskName":    "Deep work — API refactor",
  "startTime":   "09:00",               // 24h "HH:MM"
  "endTime":     "11:00",
  "category":    "focus",               // enum: focus | meeting | personal | health | learning
  "notes":       "Ship auth endpoints",  // optional, ≤ 240 chars
  "isCompleted": false,
  "createdAt":   "2026-07-02T09:14:22.031Z"
}

// The full store — a single array under one key
localStorage["chrono.schedule.v1"] = JSON.stringify([task1, task2, ...])
Design choice — times as strings, not Date objects. "09:00" is trivially sortable, JSON-safe, timezone-free, and cheap to render. We only convert to minute-integers when we need arithmetic (positioning tasks in the grid).

Persistence pattern (LocalStorage)

Three functions is all it takes. Wrap reads in try/catch so a corrupted blob can't brick the app.

const STORAGE_KEY = "chrono.schedule.v1";

// Read once on boot
function loadTasks() {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    const parsed = raw ? JSON.parse(raw) : [];
    return Array.isArray(parsed) ? parsed : [];
  } catch {
    return [];  // corrupted? start fresh
  }
}

// Write after every mutation
function saveTasks(tasks) {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(tasks));
}

// Optional: cross-tab sync
window.addEventListener("storage", (e) => {
  if (e.key === STORAGE_KEY) rerenderAll();
});

Namespace the key (chrono.schedule.v1). The v1 suffix lets you migrate the schema later without eating existing users' data — write a one-time migrateV1toV2() when the shape changes.

03

Step-by-Step Implementation Plan

Ship in vertical slices, not horizontal ones. Get a rough end-to-end path working first (add → render → persist), then loop back and polish. This order stays motivating because you have something to click at every stage.

Semantic HTML skeleton

Write the full page structure with real tags — <aside>, <main>, <header>, <section>, <form>. No styling yet. Include the modal markup (hidden by default). Add ARIA labels to icon-only buttons.

Day 1 · 1 hour

Design tokens as CSS variables

Declare all colors, radii, fonts, and shadows on :root. Import Space Grotesk + JetBrains Mono. Now the aesthetic is systemic — changing one variable ripples everywhere.

Day 1 · 1 hour

Cinematic background stack

Layer five position: fixed divs: gradient, three blurred neon orbs, subtle SVG-noise. Add a CSS keyframe (@keyframes drift) so the orbs float. This is 60% of the "premium" feel.

Day 1 · 45 min

Glassmorphism base class

Build one .glass utility: translucent gradient background, backdrop-filter: blur(24px), inset white highlight, outer drop shadow. Apply to sidebar, stat cards, grid wrapper, and modal.

Day 1 · 30 min

Neon pill button component

This is the hero interaction. Layer inset 1px rgba(255,255,255,0.3) on top, 0 0 24px cyan outside, gradient fill in between. Add :hover lift + boosted glow. Make cyan (primary) and red (danger) variants.

Day 1 · 45 min

Static task list (no state yet)

Hard-code 3–4 dummy task cards in HTML. Style them until they feel right — accent bar, circular check, ellipsis on long names. Get the visual language locked before adding logic.

Day 2 · 45 min

Add-task flow — modal + form + array push

Wire "New Task" to show the modal. On submit, push a new object into a JS array, call renderGrid(), close the modal. No LocalStorage yet — just get the round-trip working in memory.

Day 2 · 1 hour

Delete + toggle-complete

Attach click handlers via event delegation on #grid. Distinguish .task-check clicks (toggle isCompleted) from card clicks (open edit modal). Re-render after every mutation.

Day 2 · 45 min

LocalStorage persistence

Add loadTasks() on boot and saveTasks() after every mutation. Refresh the page — your data should still be there. This is the moment the app stops being a toy.

Day 3 · 30 min

Time-block positioning

Replace the flat list with the grid. Convert startTime/endTime to minutes, compute pixel top and height based on hour row height. Add the pulsing "now" line at currentMinutes / 60 × HOUR_HEIGHT.

Day 3 · 1.5 hours

Filters, search, stats, streak

Category chips + search input drive a filteredTasks() function. The stat strip (done %, hours scheduled, next up) is derived from the filtered array — pure functions, no extra state.

Day 3 · 1 hour

Polish — animations, toasts, keyboard shortcuts

Modal fade-in transform, task-complete cyan pulse, toast on every mutation. Bind ⌘K to search focus and ⌘N to open the modal. Test the mobile layout at 375px.

Core CRUD — the whole surface

// ---------- State ----------
let state = { tasks: loadTasks(), filterCat: "all", search: "" };

// ---------- CREATE ----------
function addTask(data) {
  const task = {
    id: uid(),
    taskName: data.taskName.trim(),
    startTime: data.startTime,
    endTime: data.endTime,
    category: data.category,
    notes: (data.notes || "").trim(),
    isCompleted: false,
    createdAt: new Date().toISOString(),
  };
  state.tasks.push(task);
  saveTasks();
  renderAll();
}

// ---------- UPDATE (edit) ----------
function updateTask(id, data) {
  const t = state.tasks.find(x => x.id === id);
  if (!t) return;
  Object.assign(t, data);
  saveTasks();
  renderAll();
}

// ---------- DELETE ----------
function deleteTask(id) {
  state.tasks = state.tasks.filter(x => x.id !== id);
  saveTasks();
  renderAll();
}

// ---------- TOGGLE ----------
function toggleTask(id) {
  const t = state.tasks.find(x => x.id === id);
  if (t) { t.isCompleted = !t.isCompleted; saveTasks(); renderAll(); }
}

// ---------- Boot ----------
renderAll();

Positioning a task on the time-block grid

const START_HOUR = 6;
const HOUR_H = 70;   // px per hour row

function toMinutes(hhmm) {
  const [h, m] = hhmm.split(":").map(Number);
  return h * 60 + m;
}

function placeTask(task) {
  const startMin = toMinutes(task.startTime);
  const endMin   = toMinutes(task.endTime);
  const offset   = startMin - START_HOUR * 60;
  const duration = endMin - startMin;

  const top    = (offset / 60) * HOUR_H;
  const height = Math.max(38, (duration / 60) * HOUR_H - 4);

  return { top, height };
}

Event delegation — one listener, all clicks

grid.addEventListener("click", (e) => {
  const card = e.target.closest(".task");
  if (!card) return;

  const id = card.dataset.id;
  if (e.target.closest(".task-check")) {
    toggleTask(id);          // clicked the circle
  } else {
    openModal(id);           // clicked the card
  }
});
Why event delegation? You never have to re-bind listeners when tasks are added or removed. One handler on the parent stays alive forever and uses e.target.closest() to figure out what was clicked.

Live "now" line

// Runs every 30 seconds — cheap
setInterval(() => {
  const d = new Date();
  const mins = d.getHours() * 60 + d.getMinutes();
  const offset = mins - START_HOUR * 60;
  nowLine.style.top = `${(offset / 60) * HOUR_H}px`;
}, 30_000);
